# 세션 요약: SQLite → MySQL 전환 및 환경변수 분리

- **날짜**: 2026-03-01 19:30
- **프로젝트**: Running Manager
- **브랜치**: main

## 수행 작업

### 1. SQLite → MySQL 전환
- SQLAlchemy 모델의 `String` 컬럼에 MySQL 필수 길이 지정 (`String(255)`, `String(200)`)
- `database.py`에서 SQLite URL 제거, 개별 환경변수(`DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`, `DB_NAME`)로 MySQL 접속 URL 조합
- `quote_plus`로 비밀번호 내 특수문자(`@` 등) URL 인코딩 처리
- `pool_pre_ping=True`, `pool_recycle=3600` 커넥션 풀 설정 추가
- `pymysql` (pure Python MySQL 드라이버) 의존성 추가
- `main.py`에서 SQLite 전용 ALTER TABLE 마이그레이션 코드 제거

### 2. 환경변수 파일 구조화
- `.env.sample` (커밋 대상, 템플릿), `.env.local` (로컬 개발), `.env.prd` (프로덕션) 3파일 구조
- `.gitignore`에 `!.env.sample` 예외 추가, `data/` 디렉토리 무시 추가
- `python-dotenv` 추가하여 `database.py`에서 `.env.local` 자동 로드 (IDE에서 환경변수 없이 실행 가능)

### 3. Docker Compose MySQL 서비스 추가
- MySQL 8.0 컨테이너 + healthcheck + utf8mb4 설정
- named volume 대신 호스트 바인드 마운트 (`MYSQL_DATA_DIR`, `UPLOAD_DATA_DIR`) 적용하여 Docker 재설치 시에도 데이터 유지
- backend의 `DB_HOST=mysql`로 Docker 내부 서비스명 override
- `UPLOAD_DIR`(백엔드 코드용)과 `UPLOAD_DATA_DIR`(Docker 볼륨 마운트용) 분리

### 4. 데이터 마이그레이션
- SQLite `sql_app.db` → 호스트 MySQL 마이그레이션 (users 6, activities 15, laps 176, races 1)
- SQLite → Docker MySQL 마이그레이션 (docker exec 경유)

### 5. requirements.txt 자동 생성 전환
- 수동 관리에서 `uv pip compile pyproject.toml -o requirements.txt`로 자동 생성 방식 전환
- pre-commit hook이 `uv export`로 자동 갱신
